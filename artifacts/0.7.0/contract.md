# AOHYS Development System contract

Contract version: `0.7.0`

## Three-pilot review candidate

NutriPlan, The Barber Central, and AOHYS are the only repositories authorized for this rollout. Each pilot must pass the same observable sequence: read-only audit, explicitly confirmed normalization, repository validation, live harness evidence, review and QA selection, preview readiness, recovery evidence, and a private Local Visual Recap. Product domain language, stack, commands, visual design, branch policy, release train, and paid-service state remain product-owned.

Repository residue is contextual rather than keyword-only. Explicit exclusions, tracker/workspace references, and product-scoped Impeccable integration are allowed and reported separately. Instructions that copy another product's domain or release rules, or adopt Impeccable as a global product template, remain blocking residue.

Monorepo readiness detects Convex below the repository root, accepts an existing read-only local preview alias such as `cloudflare:local`, and groups legacy physical copies by declared skill name when at least one supported Codex, T3Code, or Factory path is discoverable. File presence still does not prove runtime loading or behavioral influence.

Prepared status means the repository has its Development System adapters, required review/validation/QA/preview commands, no blocking foreign-product residue, and no logical skill that is inert across every supported harness. Operational acceptance remains separate: live observations must bind the current repository fingerprint and skill bytes to a read-only harness command, runtime version, successful exit, empty side effects, and a behavioral signature absent from the request.

The candidate ships with a three-pilot rollout and recovery playbook. Comparable evidence records the canonical system commit, product commit, audit fingerprint, managed mutation scope, preserved product files, commands, harness states, checks, PR, preview, residual risks, rollback, and gate decision for each pilot. Evidence must explicitly prove that Escuela 360 was neither inspected as a reference nor modified.

## Retained guarantees

Versions `0.1.0` through `0.6.0` remain byte-immutable. Version `0.7.0` retains reversible canonical installation, explicit skill provenance, persisted lifecycle gates, observable multi-harness parity, capability-based benchmark mappings, Implement Preview's private human gate, bounded repository fingerprinting, and fail-closed runtime preflight.

This contract authorizes neither merge nor promotion. The review candidate stops at feature branches, pull requests, reviewable previews, Local Visual Recaps, and `ready-for-human`. Merge, release, canonical HOME synchronization, production, paid activation, destructive cleanup, and rollout to Escuela 360 or any other repository require later explicit authorization.
