# Flow Research

Use official documentation, source code, specifications, first-party APIs, or research papers. Trace each material claim to the source that owns it and mark version or date sensitivity.

Inside a Git repository, load `coding-orchestration` and use its `docs_researcher` role when delegation improves speed or context quality. Dispatch through the active harness adapter. For a small question, research directly.

Return:

- claim and supporting source;
- version/date applicability;
- confirmed fact versus inference;
- unresolved uncertainty;
- consequence for the current decision.

Write a cited Markdown artifact only when the user asks for durable documentation or the active workflow requires it. Follow the repository's existing location and naming convention.
